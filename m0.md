### [Component] Execution Engine (Heuristics & Joins)

#### [MODIFY] [node.js](file:///home/etudiant/OneDrive/Documents/Master-ICO/S2_ICO/TER/vers_dangue/main/node.js)
- **Heuristics**: Implement a clause prioritization system:
  1. Clauses with 1 constant (e.g., `$x r_isa animal`).
  2. Clauses with 2 variables (e.g., `$x r_can_eat $y`) - these should be treated as "Verification" clauses if both variables are already bound, or "Exploration" if only one is bound.
- **Multi-Variable Joins**: 
  - Enhance [executerArbre](file:///home/etudiant/OneDrive/Documents/Master-ICO/S2_ICO/TER/vers_dangue/main/node.js#153-216) to handle [($x r_can_eat $y)](file:///home/etudiant/OneDrive/Documents/Master-ICO/S2_ICO/TER/vers_dangue/main/test_terminal.js#3-24) by performing a membership check if the relation exists between resolved candidates of `$x` and `$y`.
- **Pattern Matching**: Implement local filtering for queries using patterns like `$x = ba%`.
